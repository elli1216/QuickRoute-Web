# quick-route-web — Monorepo

Two independent packages:

| Package | Path | Stack | Port |
|---------|------|-------|------|
| Mock Server | `backend/` | Spring Boot 4.0.6 + Java 26 + Maven | 8080 |
| Frontend | `frontend/` | TanStack Start (React) + pnpm + Cloudflare Workers | 3000 |

## Commands

```bash
# Backend (from backend/)
./mvnw spring-boot:run   # start on :8080
./mvnw test              # run tests
./mvnw test -Dtest=ClassName

# Frontend (from frontend/)
pnpm dev                 # start on :3000
pnpm test                # vitest run
pnpm build               # production build
pnpm lint                # eslint
pnpm format              # prettier --write + eslint --fix
pnpm check               # prettier --check
pnpm run generate-routes # tsr generate (regenerates routeTree.gen.ts)
pnpm run deploy          # build + wrangler deploy
```

## Architecture notes

- **`routeTree.gen.ts`** is auto-generated by TanStack Router — do not edit manually, run `pnpm generate-routes` after adding routes.
- **`verbatimModuleSyntax: true`** in tsconfig — use `import type` for type-only imports.
- **Path aliases**: `#/` and `@/` → `./src/*`.
- **CSS** is imported with `?url` suffix: `import appCss from '../styles.css?url'`.
- **shadcn/ui**: `pnpm dlx shadcn@latest add <component>` (new-york style, lucide icons).
- Frontend deploys to Cloudflare Workers; secrets go through `wrangler secret put`, public vars in `wrangler.jsonc`.

## Backend

See [`backend/AGENTS.md`](./backend/AGENTS.md) for detailed backend guidance. Key updates:
- **JPA-backed** (H2 dev, PostgreSQL prod) — no longer file-based.
- Profiles: `dev` (default, H2 in-memory), `prod` (PostgreSQL via `DATABASE_URL` env).
- CORS configured for React dev server origin.
